---

## Definition

**Royal Jelly** is defined as:

> First-contact intelligence extracted directly from real-world signal, preserved with verifiable provenance, and validated through multi-layer verification without prior model contamination.

To qualify, a signal must pass **five protocol gates.**

---

## Gate 1 — First Contact Rule

This is the most important rule.

Royal Jelly must represent the **first structured capture** of a signal from reality.

The data **cannot** originate from:

- Model output
- Synthetic generation
- Derived summaries
- Internet scrape aggregations

**Allowed sources include:**

- Regulatory filings
- Sensor logs
- Transaction data
- Incident reports
- Primary research papers
- Government records

**Example:**

| Signal | Verdict |
|--------|---------|
| FAA incident report | Valid Jelly candidate |
| Wikipedia summary | Not Jelly |
| LLM generated analysis | Not Jelly |

If the signal is secondary or processed, it becomes Honey.

---

## Gate 2 — Provenance Verification

Every Jelly cell must have **traceable lineage.**

**Required metadata:**

```json
{
  "source_type": "regulatory_filing",
  "origin": "SEC EDGAR",
  "timestamp": "2026-03-14",
  "collector_agent": "Bee-CRE-02",
  "content_hash": "sha256:93d2..."
}
```

**Verification methods:**

- Cryptographic hash
- Timestamp record
- Source URL or ID
- Collector identity

Without provenance → **cannot be Jelly.**

---

## Gate 3 — Minimal Transformation Rule

Royal Jelly must remain **close to the original signal.**

**Allowed transformations:**

- Entity extraction
- Structural formatting
- Schema normalization

**Not allowed:**

- Interpretation
- Inference
- Reasoning chains

**Example — Allowed Jelly cell:**

```json
{
  "event": "regional bank tightening CRE lending",
  "entities": ["regional banks", "office owners"],
  "source": "FDIC bulletin"
}
```

**If reasoning is added:**

```
→ refinancing risk rising
→ property valuations may fall
```

That becomes **Honey**, not Jelly.

---

## Gate 4 — Cross-Source Confirmation

To prevent misinformation, Jelly must be confirmed by **signal coherence.**

**Methods:**

- Multiple independent sources
- Statistical anomaly detection
- Sensor corroboration
- Regulatory cross-checks

**Example:**

```
Signal: airline incident
Source A: FAA report
Source B: airport NOTAM
Source C: weather log
```

More corroboration = higher Jelly confidence.

---

## Gate 5 — Swarm Verification

Before entering the Jelly Vault, the signal must pass **SwarmJudge validation.**

**Validation checks:**

- Contradiction detection
- Hallucination filtering
- Schema correctness
- Entity validation

SwarmJudge is the hive auditor.
